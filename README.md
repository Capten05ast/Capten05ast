💬 Most chat apps feel simple… until you try to build one.

😌 I recently built a real-time responsive chat application using the MERN stack, focusing not just on features, but on how systems behave in real-world conditions.

💻 Tech Stack
- MongoDB, Express, React, Node.js
- Socket.io (real-time communication)
- ImageKit.io (cloud image storage)
- JWT Authentication + Middleware architecture

🚀 Key Features
- Real-time messaging using Socket.io rooms (group-based communication)
- Typing indicators (three dots) for live user activity
- Message status system (single tick / double tick)
- Group creation & management with dynamic member updates
- Profile picture upload & live updates
- Image sharing with cloud storage and cleanup handling
- Seen system using optimized seenBy logic
- Secure routes using JWT authentication & auth and upload middlewares

🧠 What actually mattered
- Maintaining sync between REST APIs and WebSocket events
- Handling socket reconnects & ensuring users rejoin correct rooms
- Preventing duplicate messages and missed broadcasts
- Fixing subtle bugs caused by ID mismatches & stale socket mappings
- Designing a system that works without refresh consistently


💡 Biggest takeaway
Real-time apps are not about sending messages they are about ensuring consistency, delivery, and synchronization across clients under all conditions.
